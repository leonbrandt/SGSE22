## 2 PHP vs. JavaScript

Einige Programmierer behaupten immer noch, dass es sinnlos sei, die Unterschiede zwischen PHP und JavaScript zu diskutieren, da beide einen unterschiedlichen Zweck bei der Website-Entwicklung erfüllen. Wenn es um die Frage PHP vs. JavaScript für die Website-Entwicklung geht, ein Teil der Antwort ist einfach: PHP ist eine serverseitige Skriptsprache und JavaScript ist eine clientseitige Skriptsprache. Mit dem Aufkommen von Node.js, Angular.js und React.js kann JavaScript jedoch auch für serverseitige Skripte verwendet werden. Daher ist PHP im Vergleich zu JavaScript ein ernsthaftes Diskussionsthema geworden.

### 2.1 PHP

PHP - Hypertext Preprocessor ist eine serverseitige Skriptsprache. Sie ist eine der grundlegenden Webtechnologien und wird von den meisten Websites verwendet. Sie ermöglicht es Entwicklern, dynamisch ausgeführte Webseiten zu erstellen. PHP enthält mehrere präzise Funktionen und Merkmale, die es PHP-Entwicklern ermöglichen, funktionsreiche und datenbankgestützte Webseiten zu erstellen. Es hilft bei der Erstellung interaktiver Webseiten auf hohem Niveau wie Bildergalerien, Anmeldeseiten usw. Da PHP ein Open-Source-Programm ist, kann es auf den meisten der heute verfügbaren Betriebssysteme wie Windows, Mac OS, Linux oder UNIX usw. ausgeführt werden. PHP verfügt über verschiedene Instanzen: Content Management Systeme (CMS) wie Word Press sowie SQL, MariaDB, Postgresql, usw.

### 2.2 JavaScript

JavaScript ist eine Programmiersprache, die oft als clientseitige Skriptsprache bezeichnet wird und im Allgemeinen in Browsern ausgeführt wird, um interaktive und dynamische Webseiten zu erstellen. Sie wird häufig für die Erstellung benutzerfreundlicher, kreativer Websites sowie für Nicht-Web-Projekte wie die Entwicklung von Spielen, mobilen Anwendungen, PDF-Dateien und mehr verwendet. Es muss nicht neu kompiliert werden, was die Zeiteffizienz erhöht. Es gibt auch fortgeschrittene serverseitige Versionen von JavaScript wie Node.js und React.js, die die Gestaltung von Websites mit mehr Funktionen als nur dem Herunterladen von Dateien ermöglichen. JavaScript verfügt auch über verschiede Instanzen: Frontend-Technologien wie Angular JS, JQuery, Reactjs, usw. sowie Serverseitige Technologien wie MongoDB, Node.js, Express.js, etc.

### 2.3 Ähnlichkeiten zwischen PHP und JavaScript

Sowohl PHP als auch JavaScript sind objektorientierte und interpretierte Sprachen. Sie sind Open-Source, plattformunabhängig und bieten Speicherverwaltungsfunktionen. Sie unterscheiden bei Variablen zwischen Groß- und Kleinschreibung, unterstützen Frameworks und benötigen zur Ausführung immer HTTP. PHP ist bekannt für seine objektorientierten Skripting-Funktionen. Andererseits kann JavaScript diese Eigenschaften mit gekapselten Funktionen nachmachen, die in PHP als lokale Variablen und Methoden erkannt werden. In der folgenden Tabelle sind die Gemeinsamkeiten zwischen PHP und JavaScript aufgeführt:

| Eigenschaften                                         | PHP | Javascript |
| :---------------------------------------------------- | :-: | :--------: |
| Objektorientiert                                      | Ja  |     Ja     |
| Einfache Verwendung von bestehendem Code              | Ja  |     Ja     |
| Beachtung der Groß- und Kleinschreibung bei Variablen | Ja  |     Ja     |
| Erfordert HTTP zur Ausführung                         | Ja  |     Ja     |
| Unterstützt Framework                                 | Ja  |     Ja     |
| Plattformunabhängig                                   | Ja  |     Ja     |
| Speicherverwaltung (Garbage Collection)               | Ja  |     Ja     |

Abgesehen von den Ähnlichkeiten zwischen PHP und JavaScript sind diese beiden Sprachen eine leistungsstarke Kombination, wenn sie zusammen verwendet werden. Viele Websites kombinieren PHP und JavaScript - JavaScript für das Frontend und PHP für das Backend, da sie eine große Unterstützung durch die Community, verschiedene Bibliotheken und eine große Codebasis von Frameworks bieten.

### 2.4 Unterschiede zwischen PHP und JavaScript

Abgesehen von den Ähnlichkeiten zwischen PHP und JavaScript haben beide unterschiedliche Vorteile und Nutzen, je nachdem wie und wann sie eingesetzt werden. Bei der Gegenüberstellung von PHP und JavaScript ist bekannt, dass PHP das führende serverseitige Skript ist und JavaScript der Herrscher über die Front-End-Webentwicklung ist.

#### 2.4.2 PHP vs. JavaScript: Verfügbarkeit

Die Verfügbarkeit spielt eine große Rolle, wenn es um den Unterschied zwischen PHP und JavaScript geht. Sie bestimmt, wem das Skript gehört und wer es ändern kann. PHP ist eine Open-Source-Sprache, und ihr Quellcode steht der Öffentlichkeit kostenlos zur Verfügung. Die Internet-Community ist Eigentümerin der Sprache und kann Änderungen an ihrem ursprünglichen Design vornehmen. Dies ist für PHP-Entwickler hilfreich, da es unzählige Unterstützungsangebote von gewöhnlichen Benutzern gibt. JavaScript ist ein offener Standard, der vom W3C und Ecma International gepflegt wird.

#### 2.4.3 PHP vs. JavaScript: Nebenläufigkeit

Nebenläufigkeit bedeutet, dass eine Aufgabe erst dann abgeschlossen ist, wenn alle Threads und Sub-Threads ihre Bearbeitung beendet haben. In diesem Fall beginnen und enden alle Threads zur gleichen Zeit. PHP verfügt über blockierende Multi-Thread-Eingabe/Ausgabe, um mehrere Aufgaben gleichzeitig mit einem systematischen Ansatz auszuführen. JavaScript hingegen, ein funktionales Skript mit einem Thread, erreicht ein ereignisgesteuertes Modell mit nicht blockierender Ein-/Ausgabe, das sicherstellt, dass alles gleichzeitig abläuft.

#### 2.4.4 PHP vs. JavaScript: Integration mit externem Code

JavaScript kann mit Technologien wie HTML, XML und Ajax kombiniert werden während PHP nur mit HTML-Code kombiniert werden kann.

#### 2.4.5 PHP vs. JavaScript: Laufzeitumgebungen

Sowohl PHP als auch JavaScript haben unterschiedliche Laufzeitumgebungen. Beide lassen sich jedoch mühelos in HTML-Code integrieren; beide benötigen unterschiedliche Interpreter, um ihren Code auszuführen. PHP ist einfach zu installieren und verwendet Zend Engine als Compiler und Runtime Engine. Die Zuverlässigkeit, Erweiterbarkeit und Leistung von Zend Engine spielen eine wichtige Rolle bei der wachsenden Beliebtheit von PHP. Mit der Einführung von Node.js als JavaScript-Laufzeitumgebung, erhält JavaScript die Möglichkeit, Aufgaben auszuführen, die andere serverseitige Programmiersprachen erledigen können, wie z. B. kontrollierter Zugriff auf Inhalte, Datenanalyse und individuelle Benutzererfahrung. Sowohl das Browser-JavaScript als auch Node.js werden von der V8-JavaScript-Laufzeit-Engine ausgeführt.

#### 2.4.6 PHP und JavaScript: Einfachheit

Der Hauptunterschied zwischen PHP und JavaScript besteht darin, wie einfach sie zu erlernen und zu verwenden sind. Im Vergleich dazu scheint JavaScript aufgrund seiner Komplexität, seiner erweiterten Funktionen und der Verwendung von Ereigniswarteschlangen, schwieriger zu erlernen zu sein. Obwohl PHP mit seiner größeren Bibliothek sehr leistungsfähig ist, ist es einfacher zu benutzen. PHP bietet eine Funktion für jede erdenkliche Operation.

#### 2.4.7 PHP vs. JavaScript: Kommentare

Kommentare sind lesbare Erklärungen, die der Programmierer während der Programmierung hinzufügt, um das Skript leichter verständlich zu machen. Interpreter und Compiler ignorieren sie im Allgemeinen. Die folgenden Codebeispiele vergleichen PHP- und JavaScript-Kommentare und zeigen, welche Sprache welche Art von Kommentaren unterstützt.

**PHP Kommentare:**

```php
<?php
// Dies wird dem Benutzer nicht ausgedruckt.

# "Und das hier auch nicht.";

/* Dies ist ein mehrzeiliger PHP Kommentar.
Nichts davon wird dem Benutzer angezeigt */

echo "Diese Nachricht wird dem Benutzer ausgedruckt.";
?>
```

**JavaScript Kommentare:**

```javascript

// Einzeiliger JavaScript-Kommentar

/* Mehrzeiliger
JavaScript-Kommentar */

alert("Diese Nachricht wird dem Benutzer ausgedruckt.");
?>
```

#### 2.4.8 PHP und JavaScript: Variablen

Es ist auch hilfreich PHP- und JavaScript-Variablen zu vergleichen, da diese Sprachen unterschiedliche Methoden zu ihrer Deklaration verwenden. In PHP gelten alle Variablen als lokal, es sei denn, sie werden als global deklariert. In JavaScript muss das Schlüsselwort "var" verwendet werden, um eine lokale Variable zu deklarieren, ansonsten wird sie standardmäßig als global betrachtet.

**PHP Variablen:**

```php
function local()
{
  $local_variable = 'local'; // Deklaration lokaler Variablen.
}

function global()
{
  global $global_variable;   // Deklaration einer globalen Variable.

  $global_variable = 'global';
}
```

**JavaScript Variablen:**

```javascript
function local() {
  var localVar = "local"; // Lokale Variable mit Verwendung von "var".
}

function global() {
  globalVar = "global"; // Globale Variable, keine "var"-Deklaration.
}
```

Ein weiterer wichtiger Punkt sind die booleschen Variablen. PHP akzeptiert das Schlüsselwort "true" sowohl in Klein- als auch in Großbuchstaben. JavaScript erkennt das Schlüsselwort jedoch nur, wenn es vollständig kleingeschrieben ist. Beispiel:

**PHP Boolean Variables:**

```php
is_boolean(true); // Ausgabe: TRUE

is_boolean(True); // Ausgabe: TRUE

is_boolean(TRUE); // Ausgabe: TRUE
```

**JavaScript Boolean Variablen:**

```javascript
typeof true; // Ausgabe: 'boolean'

typeof True; // Ausgabe: 'undefined'

typeof TRUE; // Ausgabe: 'undefined'
```

Der Unterschied zwischen PHP und JavaScript wirkt sich auch auf die Groß- und Kleinschreibung aus. Obwohl beide bei der Erkennung von Variablen zwischen Groß- und Kleinschreibung unterscheiden, wird in PHP bei der Deklaration von Klassen und Funktionen nicht zwischen Groß- und Kleinschreibung unterschieden, während JavaScript auch bei diesen die Groß- und Kleinschreibung beibehält.

#### 2.4.9 PHP und JavaScript: Objekte und Arrays

Der Hauptunterschied zwischen PHP und JavaScript besteht darin, dass PHP Objekte und Arrays als unterschiedliche Einheiten mit unterschiedlicher Syntax behandelt, während in JavaScript die beiden austauschbar sind. In JavaScript kann man die Syntax von Objekten und Arrays beliebig vertauschen. Außerdem gibt es in JavaScript keine assoziativen Arrays. Assoziative Arrays, auch Maps oder Dictionaries genannt, sind ein abstrakter Datentyp, der Daten in Schlüssel-Paaren speichern kann.

**PHP Objekte und Arrays:**

```php
// Array-Definition.
$arr = array(); // Neues leeres Array.
$arr = array('a', 'b', 'c');
$arr = array('a' => '1', 'a' => '2', 'c' => '3'); // Assoziativ.

// Objekt-Definition.
$obj = new stdClass(); // Neues leeres Objekt.

$obj->a = '1';
$obj->b = '2';
$obj->c = '3';
```

**JavaScript Objekte und Arrays:**

```javascript
// Array-Definition.
var arr = new Array(); // Neues leeres Array.
var arr = new Array('a', 'b', 'c');

// Array-Definition. (kurzschriftlich, gebräuchlicher).
var arr = []; // Neues leeres Array.
var arr = ['a', 'b', 'c'];

// Objekt-Definition.
var obj = {}; // Neues leeres Objekt.
var obj = {
   a: '1',
   b: '2',
   c: '3‘
 };
```

## 3 Entwicklung von Websites mit JavaScript und React.js

React ist eine JavaScript-Bibliothek, die zur Erstellung von Front-End-Benutzeroberflächen verwendet wird. Das Open-Source-Projekt wurde ursprünglich von Facebook entwickelt und ist eine der beliebtesten JavaScript-Bibliotheken. Das Ziel von React ist es, Entwicklern die Möglichkeit zu geben, ihre Benutzeroberflächen schnell zu erstellen. Vor React erstellten JavaScript-Entwickler Benutzeroberflächen von Hand mit Vanilla JavaScript, d.h. JavaScript ohne zusätzliche Bibliotheken oder Frameworks.

### 3.1 Eigenschaften von React

Es gibt zwei Hauptmerkmale von React, die darauf abzielen, die Erfahrung der Entwickler zu verbessern: JSX und Virtual DOM.

**JSX**

Das Herzstück einer jeden Webanwendung sind HTML-Dokumente (Hypertext Markup Language). HTML ist das Formatierungssystem, das zur Strukturierung einer Webseite und ihres Inhalts verwendet wird. Beim Lesen von HTML erstellen die Browser ein so genanntes Document Object Model (DOM). Das DOM sieht ein HTML-Dokument als einen Baum von Knoten, wobei jeder Knoten ein HTML-Element ist.

Heutzutage ist der Webinhalt dynamisch. Er ändert sich je nach Verhalten, Vorlieben und Interessen des Benutzers. Mit JavaScript kann man das DOM ändern, um dynamische Inhalte hinzuzufügen. Mit JSX (JavaScript eXtension) kann man das DOM mit HTML-ähnlichem Code ändern. Dies ist einer der Gründe, warum React so beliebt ist. JSX ist einfacher zu schreiben und zu verstehen als OG Vanilla JavaScript Elemente. JSX führt auch zu erheblichen Leistungsverbesserungen und ist mit allen Browsern kompatibel. Es sorgt auch für mehr Effizienz im Entwicklungsworkflow.

**Virtuelles DOM**

Um das DOM zu aktualisieren, ohne dass der Benutzer die Seite aktualisieren muss, um die Änderungen zu sehen, muss das gesamte DOM neu geladen werden. Dies ist sehr ineffizient, aber ohne diese Funktion wird der Benutzer eine schlechte Erfahrung machen. An dieser Stelle kommt das virtuelle DOM ins Spiel. Wenn die Seite JSX-Elemente verwendet, erstellt React eine Kopie des DOM, genannt Virtual DOM.

Dieses virtuelle DOM verfolgt die Elemente, mit denen der Benutzer interagiert, und aktualisiert nur diejenigen, die relevant sind. Da nicht bei jeder Interaktion das gesamte DOM neu geladen wird, verbessert dies die Leistung.

### 3.2 React Komponenten

Eine React-Anwendung besteht aus Bausteine welche Komponenten gennant sind. Die Entwickler zerlegen die Benutzeroberfläche in wiederverwendbare Code-Bausteine.
Auf einer E-Commerce-Website kann es zum Beispiel viele einzelne Artikelseiten für jedes Produkt geben. Anstatt jede einzelne Artikelseite zu erstellen, es wird nur eine Artikelkomponente erstellt und diese auch wiederverwendet.

Jede Komponente hat ihren eigenen Satz von Eigenschaften und Funktionen. Die Daten fließen von der übergeordneten Komponente zur untergeordneten Komponente, um Werte zu definieren. In diesem Fall ist die Eltern-Komponente die Produktliste und die "Top-Level"-Komponente ist die App-Komponente.

### 3.2 Zustandsverwaltung (State)

Die Zustandsverwaltung oder State-Verwaltung ist ein wichtiges Konzept in React. Ein State ist ein Objekt, das eine Reihe von beobachtbaren Eigenschaften speichert, die das Verhalten der Komponente steuern. Wenn der Benutzer mit der Benutzeroberfläche interagiert, können diese Informationen sich ändern.

Um auf das Beispiel einer E-Commerce-Website zurückzukommen: Beim Durchsuchen einer Produktliste erstreckt sich die Liste über mehrere Seiten und ermöglicht die Sortierung nach Werten wie Artikeltyp, Preis und Farbe. Wenn der Benutzer eine zweite Seite aufruft und beschließt, dass er nur blaue Artikel sehen möchte, werden die Paginierung und die Sortierungsinformationen im State gespeichert.

### 3.3 Props

Props, kurz für Eigenschaften(Properties), ermöglichen es Benutzern, Daten oder Argumente an React-Komponenten zu übergeben.Props helfen dabei, eine React-Komponente dynamischer zu machen, sind schreibgeschützt und können nicht geändert werden obwohl sich der Zustand ändern kann.
React ermöglicht dem Entwickler die Wiederverwendung von Code bzw. von Komponenten, aber im Allgemeinen werden nicht für jede Komponente die gleichen Daten benötigt.

Um auf die Seite mit der Produktliste zurückzukommen, muss der Inhalt der Komponente je nach dem vom Benutzer ausgewählten Produkt geändert werden. Hier kommen die Props ins Spiel. Wenn der Benutzer auf ein Produkt auf der Produktlistenseite klickt, wird die Komponente für den Produktartikel angezeigt. Diese Komponente Artikelliste gibt Props an ihre untergeordnete Komponente Produktartikel weiter. Die Komponente hat dieselbe Struktur, aber ihr Inhalt ändert sich in Abhängigkeit von ihren Eigenschaften. Diese Props können Daten wie Preis, Farbe oder Verfügbarkeit sein.

## 4 Entwicklung von Websites mit WordPress und PHP

Seit seiner ersten Veröffentlichung im Jahr 2003 hat sich WordPress von einer einfachen und leicht zu bedienenden Blogging-Plattform zum beliebtesten und am weitesten verbreiteten Open-Source-Website-Builder und CMS (Content Management System) entwickelt. Die Einfachheit von WordPress liegt in seinem WYSIWYG-Builder und seinem modularen Design mit Vorlagen. WYSIWYG steht für "what you see is what you get". Dieses Format für die Web-Entwicklung funktioniert über ein erkennbares Editor-System, das an Microsoft Word und andere beliebte Textverarbeitungsprogramme erinnert. Diese Art der Website-Gestaltung ist heute alltäglicher, aber zu der Zeit, als sie entwickelt wurde, stellte sie eine bedeutende Demokratisierung der Webentwicklung dar.

Heute macht die WordPress-Plattform 43 % aller Websites aus und hat einen Anteil von 65 % am CMS-Markt[3]. Jede dieser WordPress-Websites basiert auf einem Theme, bei dem es sich um eine von drei Arten handelt - ein individuelles "custom" WordPress-Theme, ein Premium-Theme oder ein kostenloses Theme. Kostenlose Themes haben in der Regel nur begrenzte Anpassungsmöglichkeiten und erhalten weniger Unterstützung für neue WordPress-Updates und benutzerdefinierte Plugin-Updates, wodurch die Website besonders anfällig für Hackerangriffe und Spam ist. Premium-WordPress-Themes sind solche, die von einem Entwickler entwickelt und aktualisiert und zum Verkauf angeboten werden.

Die Entwicklung von WordPress-Plugins wie WooCommerce für den elektronischen Handel, Yoast für die Suchmaschinenoptimierung(SEO - search engine optimization), Akismet für die Spam-Blockierung und Jetpack für Sicherheit und Website-Analyse haben die Geschäftsfunktionen von WordPress weiter verbessert. Die Plattform verfügt über ein robustes Backend, das es professionellen Webentwicklern ermöglicht, den Code anzupassen, Funktionen hinzuzufügen und die Funktionalität zu optimieren, um einzigartige Websites zu erstellen.

### 4.1 WordPress Custom Themes

Ein custom WordPress-Theme ist ein Theme, das von Grund auf für eine bestimmte Website erstellt wurde, oder ein Theme, dessen Code auf der Grundlage einer bestehenden Vorlage geändert wurde. Es wird in der Regel von einem WordPress-Webentwickler für einen bestimmten Kunden erstellt. Die meisten dieser Anpassungen werden vorgenommen, um das Design an das Branding eines Unternehmens anzupassen, gewünschte Funktionen zu implementieren, den Backend-Code zu optimieren und Funktionen zu erstellen, die dem Kunden dienen.

Durch die Verwendung benutzerdefinierter Module und die direkte Bearbeitung des Website-Codes kann ein WordPress-Entwickler das volle Potenzial von WordPress ausschöpfen. Das Backend kann so angepasst werden, dass Plugins in bestehende Geschäftssysteme und -ressourcen integriert werden können, z. B. in Bestandsverwaltungssysteme, CRMs, Verkaufsstellen, Callcenter, Kundendienstabteilungen, Social-Media-Verwaltung und vieles mehr.

PHP-, HTML- und CSS-Kenntnisse sind die Grundvoraussetzung für die Entwicklung von Websites mit benutzerdefinierten Themen unter Verwendung von WordPress. Je höcher die Kenntnisse in diesen Sprachen sind, desto mehr kann ein Entwickler die zu entwickelnde Website an die Bedürfnisse seiner Kunden anpassen. Das bedeutet, dass selbst Programmierer mit grundlegenden Kenntnissen und Erfahrungen, mit der Entwicklung benutzerdefinierter Themes in WordPress beginnen können. Man kann mit der Installation von WordPress beginnen, lokal oder auf einem Server. Ein WordPress-Theme besteht aus der Vorlagendatei index.php und einem Stylesheet, und alles andere ist optional.

**Style.css**

Das Stylesheet definiert das WordPress-Theme und muss sich im Stammverzeichnis des Themes befinden. Es ist wichtig, dass das Haupt-Stylesheet den Namen `style.css` trägt. Als nächstes müssen einige Zeilen mit Kommentaren in die Datei eingefügt werden. Hier ist ein Beispiel aus dem Stylesheet des Themes für die Firma Millennials Digitaldesign:

```css
/*
Theme Name: Millennials
Text Domain: midi
Version: 1.0.0
Description: This is the personal theme of Millennials.

Author: Millennials Digitaldesign GmbH & Co. KG
Author URI: https://millennials.team/

License: GNU General Public License v3
License URI: http://www.gnu.org/licenses/gpl-3.0.html
*/
```

**Template file**

Das WordPress-Theme muss die Hauptvorlage `index.php` enthalten. Diese Vorlage ist die Standardvorlage des Themes, und wenn keine andere Vorlage verwendet wird, greift es auf diese zurück. Bei der Entwicklung eines eigenen WordPress-Themes gibt es eine ganze Template-Hierarchie[4]. Jede Vorlage enthält normalerweise eine header.php und eine `footer.php`, die auf allen Vorlagen/Seiten gleich aussehen.

Ein Beispiel für eine `header.php`-Datei:

```php
<?php
/**
 * The template for displaying the default header
 */
?>

<!DOCTYPE html>
<html <?php language_attributes();?>>

<head>
  <meta charset="<?php bloginfo('charset');?>" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <link rel="profile" href="https://gmpg.org/xfn/11" />
  <?php wp_head();?>
</head>

<body <?php body_class();?>>
  <?php wp_body_open();?>

  <span class="scroll-anchor" id="start"></span>
  <?php get_template_part('template-parts/navbar/navbar', 'default', $args);?>

  <?php get_template_part('template-parts/header/header', 'default', $args);?>
```

`language_attributes()` gibt den Sprachtyp für den <html>-Tag aus. `wp_head()` gibt die Javascript-Links und andere Header-Funktionen aus. `body_class()` zeigt die Klassennamen für das body-Element an. `get_template_part()` bietet einen einfachen Mechanismus für Themes, um wiederverwendbare Codeabschnitte im Theme zu laden. Alle oben genannten Tags und viele weitere sind im WordPress Codex[5] gut dokumentiert.

`footer.php` ist eine einfachere Datei. Sie enthält benutzerdefinierte Inhalte und eine `wp_footer()`-Funktion, die WordPress häufig für Plugins und Skripte verwendet.

```php
<?php
/**
 * The template for displaying the default footer
 */
?>

<?php get_template_part('template-parts/footer/footer', 'default', $args);?>
<?php get_template_part('template-parts/content/content', 'contact-offcanvas', $args);?>
<?php wp_footer();?>

</body>

</html>
```

Und schließlich die Hauptvorlagendatei `index.php`:

```php
<?php
/**
 * The main template file
 */

// REDIRECT TO ROOT
wp_redirect(get_site_url(), 301);exit;
?>

<?php get_header();?>

<div id="content">
    <?php if ( have_posts() ) : while ( have_posts() ) : the_post(); ?>
    <?php the_content(); ?>
    <?php endwhile; else: ?>
    <p>No posts.</p>
    <?php endif; ?>
</div>

<?php get_footer();?>
```

Die `index.php` enthält die Dateien `header.php` und `footer.php` mit den Funktionen `get_header()` und `get_footer()`, und dasselbe gilt für die Fußzeile. Dann gibt es eine WordPress-Schleife, die nach den Beiträgen sucht und den Inhalt für jeden einzelnen ausgibt. So sieht das Theme für Millennials Digitaldesign GmbH im WordPress-Backend aus:

## 5 Fazit

React.js ist eine gute Option für JavaScript-Entwickler, die Single-Page-Webanwendungen erstellen. Es ermöglicht Entwicklern, komplexe Websites mit wiederverwendbaren Komponenten zu erstellen. Ein großer Vorteil ist die Möglichkeit, nur die Komponente mit den relevanten Daten zu aktualisieren. Anstatt die gesamte Seite neu zu laden, verfolgt React über sein virtuelles DOM genau, was sich geändert hat. React ist schnell, skalierbar und flexibel, da es sich um eine Bibliothek und nicht um ein Framework handelt. Außerdem gibt es eine große und aktive Entwicklergemeinde und eine gute Dokumentation.

WordPress ist eine gute Lösung für eine breite Palette von Anwendungen. Sowohl Anfänger als auch erfahrene Entwickler können damit kleine Blogs, große E-Commerce-Sites und alles dazwischen erstellen. Wenn ein Entwickler über PHP-Erfahrung verfügt, egal auf welchem Niveau, ist WordPress und insbesondere die benutzerdefinierte Theme-Entwicklung mit WordPress eine gute Wahl, um mit der Entwicklung von Websites zu beginnen.

Eine eindeutige Meinung darüber, welches Tool für die Entwicklung von "Content" besser ist, kann nicht geäußert werden. Sowohl WordPress als auch React.js verfügen über unterschiedliche Funktionalitäten und Frameworks und bieten somit unterschiedliche Möglichkeiten. Folglich müssen die Entwickler je nach ihren Anforderungen und Erfahrungen entscheiden, welches Tool sie verwenden möchten.
